# ДомКофе — дипломный проект

Сайт кофейни: **React (Vite)** + **FastAPI** + **SQLite**. В продакшене через **Docker** фронт отдаёт **nginx**, бэкенд — отдельный контейнер, загрузки картинок — общий volume.

---

## Запуск через Docker

1. Скопируй `.env.example` в `.env` и заполни:
   - `SECRET_KEY` — длинная случайная строка (для JWT админки);
   - `ADMIN_USERNAME`, `ADMIN_PASSWORD` — логин и пароль администратора;
   - при необходимости `NGINX_PORT` (по умолчанию **80**).

2. В корне репозитория (`папка с docker-compose.yml`):

```bash
docker compose up -d --build
```

3. Открой в браузере:
   - **Сайт (фронт):** `http://localhost` (или `http://localhost:<NGINX_PORT>`).

Первый запуск поднимает БД и при необходимости создаёт пользователя админки из `.env` (см. seed в backend).

## Куда заходить

| Что | Адрес |
|-----|--------|
| Публичный сайт | `/`, `/menu`, `/news`, `/contacts`, `/about` |
| Вход в админку | `/admin/login` |
| Панель после входа | `/admin` (дашборд), `/admin/items`, `/admin/news`, `/admin/messages` |

Учётные данные — те же, что в `.env` (`ADMIN_USERNAME` / `ADMIN_PASSWORD`).

---

## API (публичное)

Базовый префикс: **`/api`**. В Docker снаружи это тот же хост, что и фронт (nginx проксирует на бэкенд).

| Метод | Путь | Назначение |
|--------|------|------------|
| `GET` | `/api/menu` | Меню по категориям (позиции, варианты, цены) |
| `GET` | `/api/news` | Список новостей (query: `page`, `pageSize`) |
| `GET` | `/api/news/{id}` | Одна новость |
| `GET` | `/api/promos/latest` | Блок «последняя новость» для промо |
| `POST` | `/api/contacts` | Форма обратной связи (JSON тела) |

Проверка бэкенда внутри сети Docker: `GET /health` есть у FastAPI, но **с хоста через nginx по умолчанию не проксируется** — для отладки можно временно пробросить порт backend в `docker-compose` или смотреть логи контейнера.

## API (админка)

Все под префиксом **`/api/admin`**, кроме логина нужен **Bearer JWT** (токен с `POST /api/admin/login`).

Кратко:

| Метод | Путь | Назначение |
|--------|------|------------|
| `POST` | `/api/admin/login` | Выдача токена |
| `GET` | `/api/admin/me` | Текущий админ |
| `GET` | `/api/admin/stats` | Статистика для дашборда |
| `POST` | `/api/admin/upload` | Загрузка картинки (меню/новости) |
| CRUD | `/api/admin/categories`, `/items`, `/news` | Категории, позиции меню, новости |
| CRUD | `/api/admin/messages` | Сообщения с формы контактов |

Загруженные файлы отдаются по **`/uploads/...`** (nginx).

## Локальная разработка без Docker (кратко)

- **Backend:** из каталога `backend`, `uvicorn` на порт **8000** (как в `DATABASE_URL` / `UPLOADS_DIR` в документации проекта).
- **Frontend:** `cd coffeecafe && npm install && npm run dev` — Vite на **5173**, прокси на `/api` и `/uploads` к `http://127.0.0.1:8000` (см. `vite.config.ts`).

CORS в бэкенде разрешён для `http://localhost:5173` и `http://127.0.0.1:5173`.

---

## Данные

В Docker БД и загрузки лежат в **named volumes** (`sqlite_data`, `uploads`). Для переноса на другой ПК нужно копировать не только код, но и дамп/файлы volumes — см. комментарии в проекте или документацию по бэкапу.
